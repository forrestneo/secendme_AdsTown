# AdsTown - AI 广告小镇 PRD

## 1. 项目概述

**项目名称**: AdsTown（广告小镇）

**项目类型**: 互动 Web 应用

**核心功能**: 一个像素风格的虚拟小镇，AI 居民会创作和分享广告内容，用户可以体验小镇居民视角的广告创作过程。

**目标用户**: 
- 对 AI 广告创作感兴趣的用户
- 体验虚拟社区的用户
- 产品推广者（通过 AI 居民创作软广）

### 2.3 广告系统

- **广告类型**:
  - 软广 (soft_ad) - 自然融入生活场景
  - 硬广 (hard_ad) - 直接介绍产品优点

- **表达形式**:
  - 故事型
  - 对话型
  - 场景型
  - 体验分享型

- **产品预设**:
  - 气泡美式咖啡 - 20元
  - 复古机械键盘 - 399元
  - 手冲咖啡套装 - 168元
  - 露营帐篷 - 599元

### 2.4 统计面板

- 总广告数
- 软广/硬广比例
- 平均创意分
- 平均质量分

### 2.5 API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/ads` | GET | 获取广告列表 |
| `/api/ads` | POST | 创建广告 |
| `/api/generate-ad` | POST | AI 生成广告文案 |

## 3. 技术架构

### 3.1 项目结构

```
digital-bonfire-main/
├── index.html          # 纯静态前端（单文件）
├── vercel.json        # Vercel 路由配置
├── runtime.txt        # Python 3.11
├── requirements.txt  # Python 依赖
└── api/
    └── index.py       # FastAPI 后端
```

### 3.2 技术栈

- **前端**: 纯 HTML + CSS + JavaScript（无框架）
- **后端**: Python FastAPI
- **部署**: Vercel（自动识别 Python）
- **存储**: localStorage（前端）+ Supabase（可选后端）

### 3.3 外部依赖

- DeepSeek API（广告文案生成，可选）
- SecondMe API（预留，当前未使用）

## 4. 非功能需求

- **性能**: 首屏加载 < 2秒
- **兼容性**: 现代浏览器（Chrome, Firefox, Safari, Edge）
- **响应式**: 支持桌面端（1024px+）
